# Spec: SQLite-First Local DB

## Background

当前模板的本地开发路径默认依赖 PostgreSQL 连接。

现状带来两个直接问题：

- 本地首次启动前，开发者需要先准备数据库服务并填写连接串，启动门槛偏高
- 仓库虽然已有数据库相关命令与部分说明，但“首次启动需要做什么、初始化顺序是什么、默认本地模式是什么”没有形成一条清晰、低摩擦的路径

该模板面向单用户、自部署、可复用项目基座。对于这类模板，默认本地开发体验应优先追求零外部依赖、最短启动路径；同时，部署到 NAS/服务器时仍需保留切换到持久化数据库的简单路径。

## Goal

将模板的默认本地数据库体验改为 SQLite-first，使开发者在复制模板后无需额外安装或启动 PostgreSQL 即可完成初始化并进入开发。

同时，模板必须继续支持在部署场景中切换到 PostgreSQL，并且切换方式应足够简单，避免要求改动应用代码或手工调整多处配置。

## Target Users

- 首次使用该模板的新项目开发者
- 希望本地快速启动、避免维护数据库容器的个人开发者
- 需要在部署时切换到持久化数据库的自部署项目维护者

## User Scenarios

### Scenario 1: 本地首次启动

开发者复制模板、安装依赖、复制环境变量样例、执行数据库初始化命令后，即可在本地启动应用，无需先安装、启动或连接 PostgreSQL。

### Scenario 2: 本地迭代开发

开发者在本地反复重启开发环境时，默认数据库数据应继续保留在本地 SQLite 文件中，不要求每次重新初始化。

### Scenario 3: 部署前切换数据库

开发者准备将应用部署到 NAS 或服务器时，应能够通过一组明确的环境变量切换到 PostgreSQL，而不需要改动业务代码或调整调用方式。

### Scenario 4: 新用户理解启动路径

第一次接触模板的开发者阅读 README 或数据库文档后，能够清楚知道：

- 本地默认使用什么数据库
- 首次启动需要执行哪些命令
- 何时需要切换到 PostgreSQL
- 切换时需要修改哪些配置

## Functional Requirements

### 1. Default Local Database Mode

- 模板必须将本地默认数据库模式定义为 SQLite。
- 模板默认本地启动路径不得要求预先安装、启动或连接 PostgreSQL。
- 模板必须提供可直接使用的本地默认数据库配置样例。
- 本地默认数据库必须使用文件持久化方式，而不是仅内存模式。

### 2. Initialization Path

- 模板必须提供明确的首次数据库初始化路径。
- 模板必须提供一个面向首次使用者的数据库初始化命令或等价入口，用于完成本地数据库准备所需步骤。
- 模板必须说明初始化命令的用途、执行时机和成功后的预期结果。
- 模板必须说明 seed/管理员账号初始化如何触发。

### 3. Deployment Database Switching

- 模板必须继续支持 PostgreSQL 作为部署场景数据库。
- 模板必须支持通过环境变量切换数据库模式。
- 从本地 SQLite 切换到 PostgreSQL 时，应用层使用方式必须保持一致，不要求调用方改代码。
- 切换到 PostgreSQL 的路径必须足够简单，达到“一键切换”目标。
- “一键切换”在本 spec 中定义为：开发者只需切换约定环境变量并执行同一组标准数据库命令，即可完成数据库模式切换。

### 4. Documentation

- 模板必须在 README 或等价入口文档中明确写出 SQLite-first 的默认本地开发路径。
- 模板必须补充数据库使用文档，清楚区分“本地 SQLite”与“部署 PostgreSQL”两种模式。
- 文档必须明确说明首次启动步骤、常见数据库命令、切换方式和适用场景。
- 文档必须避免让新用户误以为本地默认仍然依赖 PostgreSQL。

### 5. Environment Examples

- 模板必须提供与默认本地 SQLite 模式一致的环境变量样例。
- 模板必须提供 PostgreSQL 部署模式所需的环境变量示例，供部署时参考。
- 环境变量说明必须清楚标识哪些项是本地默认必需项，哪些项仅在 PostgreSQL 模式下需要配置。

## Non-Functional Requirements

- 本地启动路径必须尽量短，适合第一次复制模板后的快速验证。
- 数据库模式切换说明必须清晰，避免隐式规则。
- 文档必须以模板用户视角组织，而不是假设使用者已了解仓库内部数据库实现细节。
- 新方案不得降低模板作为自部署基座的可用性。

## Out of Scope

- 设计完整的数据库抽象层实现细节
- 规定具体 ORM 方言适配方案
- 规定具体 migration 文件组织方式
- 引入默认内存数据库作为主开发模式
- 自动完成 SQLite 到 PostgreSQL 的数据迁移
- 重构与本需求无关的认证、页面或业务功能

## Acceptance Criteria

- 新用户按 README 的默认本地启动步骤操作时，无需额外准备 PostgreSQL 即可启动模板。
- 模板文档明确表述本地默认数据库为 SQLite 文件库。
- 模板提供明确的数据库初始化入口，且首次使用者可以据此完成初始化。
- 模板提供明确的 PostgreSQL 切换说明，且说明中不要求改动应用代码。
- 模板环境变量样例能够分别支撑本地默认模式与部署切换场景。
- spec 中不存在会阻塞后续规划的关键歧义。
